# 🔧 Исправление порядка элементов в модальном окне на мобильных

## ✅ Что сделано

### Проблема
На мобильных устройствах в модальном окне **детали** отображались сверху, а **фото** снизу.

### Решение
Использовал **Flexbox с `order`** вместо CSS Grid для мобильных устройств.

### Технические детали

#### До изменений:
```css
/* Не работало корректно */
.modal-body {
  grid-template-areas: "gallery" "details";
}
```

#### После изменений:
```css
/* Mobile devices (< 768px) */
@media (max-width: 768px) {
  .modal-body {
    display: flex !important;
    flex-direction: column !important;
    grid-template-columns: unset !important;
    grid-template-areas: unset !important;
  }
  
  .modal-gallery {
    order: 1 !important;  /* Фото СВЕРХУ */
  }
  
  .modal-details {
    order: 2 !important;  /* Детали СНИЗУ */
  }
}
```

### Как это работает

**Desktop (> 768px):**
```
┌──────────┬──────────┐
│  Gallery │ Details  │  ← Grid: 2 колонки
│  (Фото)  │ (Инфо)   │
└──────────┴──────────┘
```

**Mobile (< 768px):**
```
┌──────────────┐
│   Gallery    │  ← order: 1 (СВЕРХУ)
│   (Фото)     │
├──────────────┤
│   Details    │  ← order: 2 (СНИЗУ)
│   (Инфо)     │
└──────────────┘
```

## 💡 Почему используется `!important`

```css
order: 1 !important;
```

**Причины:**
1. Grid на desktop имеет высокую специфичность
2. Нужно гарантированно переопределить для мобильных
3. `!important` обеспечивает правильное отображение

**Альтернатива без `!important`:**
- Потребовалось бы изменять HTML структуру
- Или использовать JavaScript для перестановки элементов
- Текущее решение — самое простое и надёжное
